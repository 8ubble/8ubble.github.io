原来我是一只并不出名的逼乎网猴，然后现在正在试运行GitHub找找啥叫“另一个网盘”。

这几天捣鼓hexo的时候踩了很多坑，现在就大概讲一下怎么配一个自定义博客。

    1. 你得有一个GitHub

我的Github开了应该有两年了，不过最近才开始学会使用这个……

代码托管的挺方便的这是其一，其二是因为我今年在上找到了不少的CTF的资料，基本上是以hexo+Github为主，其三比较方便参（chao）考（xi）代码（P.s. 建议别再用网盘啦，用Github显得多高大上，当然你开个知乎专栏写一个也可以233）

    1. 开GitHub.io需要注意什么？

其实我应该叫他“GitHub Pages”，

GitHub Pages is designed to host your personal, organization, or project pages from a GitHub repository.

先创建一个仓库，然后名字是xxx（英文名，你想让你博客叫什么名就取什么英文名）.gitHub.io之类的，比如说像这样：

一个账户好像只能开一个个人或组织的page 吧，项目的如果没有自定义域名是默认放在当前的gitHub.io下的，具体请看官方文档

然后如果你是用Windows更新的话，下个git-scm 之类的，我的电脑是Windows 10然后用不了官方推荐的GitHub for Windows（大概是我被墙了所以我用不了），下完之后会有Git CMD、Git BASH、Git GUI，留着备用。

你这样也可以：

git clone也可以，比较推荐用git CMD或者是git Bash来进行git clone https://github.com/username/username.github.io ，

然后找到你刚刚克隆下来的GitHub.io的仓库，找到index.html，进行相应的更改之后再进行Commit和Pull即可。

或者你也可以在项目管理页那儿进行编辑。

    1. hexo配置的坑

以它最坑爹。

安装node,js之后，然后找到对应主题进行clone，注意哦，都是免费的。

或者你自己可以DIY一个。

然后在themes文件夹里复制你刚刚的主题文件夹，是解压主题之后的文件夹。然后在博客根目录的_config.tml里开始进行配置。有些地方是不能留下空格的（我直接找了一份别的博客的_config.yml稍微修改了一下，避免了这个坑），然后保存。里面的permalink 就是你博客文章的目录结构，可以用pretty来简单的设置成日期+文章标题.html，也可以用自己喜欢的结构来设置。 记得把encoding 设置成utf-8，这样有利于中英文双语的写作和阅读。

然后你可以试一下hexo sever -p 5000，看本地博客能不能跑起来。我在这儿选用了4000端口是因为我的电脑上有个别的程序使用了4000端口。确认无误后，可以用hexo deploy开始完工了。

到这里你就可以开始写博客了，hexo new "My New Post"所有的文章直接放在_posts文件夹下面，格式就是我们之前提到的markdown文件，默认的格式是.md和.markdown文件。每篇文章的开始处需要使用yml格式来写明这篇文章的简单介绍，格式如下：
